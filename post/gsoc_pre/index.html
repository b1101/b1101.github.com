GSoC 2016 - prelude

May 17, 2016   #gsoc  #browsix  #go 

My Google Summer of Code proposal got accepted and I am going to write code for Browsix all summer this year. Bobby Powers is going to be my mentor. And if all of that was not ‘cool’ enough - I’m doing it all in Go.

GSoC oficially starts on May 23 and it is one of the main reasons that made me create this blog

A not-so-short intro of myself

Text beneath this line is basically an unnecessary description of myself and how I applied/got accepted I’ve been trying, installing, breaking, reinstalling, breaking, configuring, breaking, compiling and breaking different Linux(*BSD) distributions - Arch, Void, Exherbo, Gentoo, Funtoo, Guix, Nix, you name it…(I even installed Stali, but I was too lazy to set it up) for the last ~4 months

I was so into ‘distrohopping’ that I even got to a point when I had an exam at 8.45 at the morning and I skipped almost all the lectures in the subject and was totally not ready, but I was in a process of installing a distro for 3 days and I just managed to successfully boot, made all the necessary hardware work and even started X. 3 days without a GUI(other than an occasional one from a LiveUSB) and I could not stop. I was configuring my desktop until about 3am, when I felt that I need to go to sleep, because of the exam. I layed in bed for about 1 hour thinking about Linux, then I got up and continued.

I might have taken it to some kind of an extreme, reinstalling a system 1,2,3 times a week(by the way each time installing a new OS I formatted my drive, thinking I know enough to make everything work the 1st time and I did not ask questions to anyone, I was to the one to find out the answers by myself), but it was an amazing learning experience. I learned an awful lot of stuff.Oh, and I do not need an install guide anymore!

GSOC - application

One day, when my ‘distrohopping spree’ got into a somewhat ‘calmer’ state I discovered Google Summer of Code. It was ~2 months until the application started, I read some articles on the Internet, stating it’s very hard to get into, you need to prepare etc. I decided to study, to learn and become a ‘real’ programmer and then ‘maybe, I would be accepted next year’. It became of a somewhat like an unreal dream of mine to participate in a project like that(I did not even really know how to code at the start of this study year)

When the organization list was announced I really wanted to know, which are there. One of the projects that got my attention was Browsix. Why? Because it seemed like something I would ultimately like to do(cmon, words like Unix and Shell are enough to get me hooked), but at the same time it seemed so hard and out-of-my-league that I would never be able to do this.

You are usually supposed to send a do some small task for the org before applying - fix some issue, send a pull request, for example. And so, as I like to dream about stuff, “Ok, so that is something I would like to do. Let’s imagine that some day in the future, when I would understand what’s actualy going on, I got on this page, what would I do?”

I went on to the pull request page. Everything seemed impossible. I saw some pull requests already having someone working on them, then I got to a command I’ve never used before, called - xargs. I honestly did not even know it exists. Moreover, the implementations of commands would be done in Typescript and an in-browser port of ‘node’ would be used to interact with the in-browser FS.(it’s worth mentioning that I had no idea what Node.js and Typescript really are at that point and I did not know much more about JavaScript, than that is uses a var to declare variables and that there are cool docs on MDN)

Long story short(it is already pretty long actualy) I managed to implement it. I sent a pull request consisting of many commits, then I had to fix it. At the end it all summed up to a terrible pull request of 18 commits. I am still ashamed of that fact, but, neither did I know how to acutally use Git(I could have just, for example, resetted head, or rebased, or cherry picked, but no - all of the terrible history of my commits went into the tree and they are not going away anymore)

Anyway, my first ever pull request was accepted and I felt happy. Much happy, many proud.

I wrote my proposal in the first 2-3 days of the application window, but I was very confused about the whole application process. After reading some of the source code of other init systems I decided to not apply this year. I felt like I am not qualified enough and sending a proposal now would be just a waste of time, it would only make me upset when I find out I am not accepted in a month. At the end I submitted my proposal several hours before the deadline. I only sent a proposalto 1 organisation and did not submit any drafts.

I still can not believe that I am one of the students participating in Google Summer of Code. It feels impossible. (Especially for a first year Bachelor student, who decided to study programming in the last semester of high school. I was in-between studying Jazz or Chemistry)

That turned out to be a pretty long blogpost, didn’t expect that. And by the way, my skillset broadened since that I now am much more knowledgable about stuff, most importantly I have a clear idea on what I have to do and how I am going to do it
